Well, of course it bails down to time. However, Marcello and I have  
agreed that our initial release shall compile with both asterisk 1.2  
and 1.4 so that future patches will only have to be applied against  
one tree. Apart from that we're ready for a release.

Beyond that we're still waiting for input by all the other commited  
people who have created other versions of chan_sccp, especially those  
working with asterisk 1.4. As we created a fork of the berlios  
chan_sccp to bring this work forward, we don't want it merely to be  
based on "our stuff" because we started it.

So on this occasion again: Please submit your patches or tell about  
your custom version of chan_sccp. We really with to re-unite all the  
useful however "stray" patches and pieces of code out there. I know  
how much work it is to actually identify the changes entered  
quick&dirty into one's own source tree, but it's definitely worth it.

Attached to this mail is the most recent version of Marcello's and  
mine joint work. I just added a feature that I had to unravel from my  
own legacy 1.2 tree: Using the global option "recorddigittimeoutchar  
= on", the '#' to terminate variable-length numbers as they commonly  
occur in European countries, is recorded to the call logs if it was  
dialed at first. So redialing works without manual post-dialing the #  
from the logs.

> Phone firmware versions - Any chance we can get agreement on the

I have found that firmware version 7.2.2 works quite stable for the  
7940 and 7960. Although the 8.0.4 version seems to be stable too, I  
did not see a noticeable benefit in using this version. Please  
correct me if I missed an important issue.

Apart from that the most recent versions should be allright since we

>
> DST/locale settings - Can anyone explain what files are needed to  
> correctly localise Cisco phone 7940/7941/7960/7961 etc. During UK  
> DST time all my 7940/7960 phones showed the correct time and my  
> 7961 phone was 1 hour out. Now that DST has ended, all my  
> 7940/7960s are 1 hour out and 7961 is correct! I would like my 7961  
> to show the date in UK format eg. d/m/y as my 7940/7960 already do  
> (taken from sccp.conf). I know that there was a change in the way  
> locale settings are used between 7940/7960 series and 7941/7961  
> series. I have access to Cisco downloads.

AFAIK the 7940/60s don't get the timezone information whatsoever.  
They just display the local time of the asterisk server, don't they?  
So you should check your server for that issue.

The 7941/61/70s all use the following settings for the timezone/ 
display issue:

     <dateTimeSetting>
         <name>CMLocal</name>
         <dateTemplate>D.M.Y</dateTemplate>
         <timeZone>W. Europe Standard/Daylight Time</timeZone>
     </dateTimeSetting>

Insert this just with a valid timezone string insted of W. Europe...  
before the <callManagerGroup> tag of your xml config. At the end of  
this mail I have compiled a list of valid timezone entries from the  
simplevoip wiki at http://voip-info.simplevoip.info/wiki/
